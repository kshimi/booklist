# プロジェクト概要

**作成日**: 2026-02-28
**ステータス**: ドラフト v0.1
**対象フェーズ**: フェーズ1（静的SPA）

---

## プロジェクト名

蔵書管理システム

---

## 背景・課題

Google Drive の `70_Book` フォルダ配下に、スキャン済み書籍 PDF 約 1,900 ファイルが保管されている。ファイルはジャンル別サブフォルダに格納されており、同一書籍について「オリジナル版 / iPad向け軽量版（`ipad3_`）/ Kindle向け軽量版（`kindlep_`）」の最大3バージョンが存在する。

書誌メタデータ（タイトル・著者・ISBN・ページ数・シリーズ名）はファイル名に独自形式で埋め込まれており、一元的な蔵書管理の仕組みが存在しない。

| # | 課題 |
|---|------|
| 1 | 蔵書全体を横断的に検索・閲覧する手段がない |
| 2 | 同一書籍の複数バージョンを把握しにくい |
| 3 | ジャンル別・著者別の保有状況を俯瞰できない |
| 4 | 次に読む本を選ぶ際の指針が得られない |

---

## 目標

### フェーズ1（本プロジェクト）

1. Google Drive のファイルリスト（CSV）からメタデータを自動解析し、正規化された蔵書カタログ（`data/books.json`）を構築する
2. 蔵書を検索・閲覧・フィルタリングできる静的 SPA を提供する
3. ISBN が存在する書籍について外部書誌 API から追加情報を取得・表示する

### フェーズ2（将来）

- AI を活用した読書レコメンド機能（バックエンドサーバー導入を前提）

---

## 対象ユーザー

個人利用（蔵書所有者のみ）。パブリックアクセスは不要。

| 項目 | 内容 |
|------|------|
| 利用者 | 蔵書所有者のみ |
| 実行環境 | PC または iPad のブラウザ |
| PDF 参照 | Google Drive 上の PDF ファイルを Google Drive リンク経由で閲覧する |

---

## 主な機能

| 機能領域 | 主な機能 |
|---------|---------|
| **データ処理**（ビルド前工程） | CSV インポート、ファイル名パース、ジャンル推定、重複排除、`books.json` 生成 |
| **書籍一覧・検索** | キーワード検索（インクリメンタル）、ジャンルフィルタ、著者絞り込み、ソート、ページネーション（50件/ページ） |
| **書籍詳細** | 基本情報表示（パース済みメタデータ）、Google Drive PDF リンク、外部書誌 API 補完表示（表紙・出版社・内容紹介） |
| **統計・分析** | ジャンル別分布グラフ、著者別保有冊数ランキング（上位20名） |
