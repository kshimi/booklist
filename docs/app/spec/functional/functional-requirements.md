## 2. データ処理機能

データ処理はビルド前工程（Claude Code による一括処理）として実行する。SPAの実行時には処理済みの `data/books.json` を読み込む。

### F-1: CSVインポート・メタデータパース

#### 概要

Google Apps Script により取得した CSV ファイル（`data/booklist.csv`）を読み込み、各 PDF ファイルのメタデータを抽出する。

#### 入力

| 項目 | 仕様 |
|------|------|
| ファイル形式 | CSV（Google Apps Script 出力） |
| 想定レコード数 | 約 1,900 行 |
| 必須カラム | フォルダパス、ファイル名、ファイルサイズ(MB)、MIMEタイプ、作成日、最終更新日、ファイルURL、ファイルID |

#### 処理仕様

**対象ファイルの絞り込み**

- MIME タイプ `application/pdf` のファイルのみ処理対象とする
- 以下のファイル名を含むレコードは除外する（管理用ファイル）:
  - `self_check`
  - `20120330_1246_33_0706`
  - `to_1733_912_003_5_at`

**ファイル名パース順序**

ファイル名は以下の書式に概ね従う:

```
[バージョンプレフィックス_]タイトル（シリーズ名）著者名 ページ数p_ISBN.pdf
```

各フィールドを以下の順で段階的に抽出する:

| ステップ | 処理内容 |
|---------|---------|
| 1. バージョン判定 | `kindlep_` プレフィックス → Kindle版、`ipad3_` プレフィックス → iPad3版、なし → オリジナル版。プレフィックスをファイル名から除去する |
| 2. 拡張子除去 | 末尾の `.pdf` を除去 |
| 3. ISBN抽出 | 末尾の `_XXXXX` パターン（10〜13文字の英数字）を抽出・除去 |
| 4. ページ数抽出 | 末尾の ` NNNp` パターン（半角スペース + 数字 + p）を抽出・除去 |
| 5. シリーズ名抽出 | `（）` または `()` 内のテキストを全て抽出。複数ある場合は `／` 区切りで結合。括弧と内部テキストをファイル名から除去 |
| 6. 著者名抽出 | 最後の括弧の後ろに残るテキストを著者名とする。括弧がない場合は末尾の日本語2単語（姓 名）パターンにマッチさせる |
| 7. タイトル抽出 | 上記ステップ後に残ったテキストをタイトルとする |

#### 出力（1レコードあたり）

| フィールド | 内容 |
|-----------|------|
| `title` | パース済みタイトル |
| `author` | 著者名 |
| `isbn` | ISBN（10桁または13桁）。なければ `null` |
| `pages` | ページ数（整数）。なければ `null` |
| `series` | シリーズ名。なければ `null` |
| `version` | `original` / `kindle` / `ipad3` |
| `folder_path` | フォルダパス（ジャンル推定に使用） |
| `file_url` | Google Drive ファイルURL |
| `file_id` | Google Drive ファイルID |
| `file_size_mb` | ファイルサイズ（MB） |

### F-2: ジャンル推定

#### 概要

全書籍（重複排除前の各ファイル）に対してジャンルを割り当てる。

#### 処理仕様

フォルダパスに含まれるキーワードを以下の優先順で判定し、最初にマッチしたジャンルを採用する。

| 優先度 | マッチ条件 | ジャンル |
|--------|-----------|---------|
| 1 | パスに `エッセイ` を含む | エッセイ |
| 2 | パスに `日本の作家` を含む | フィクション（日本） |
| 3 | パスに ` SF` または `/SF` を含む（境界条件付き） | SF |
| 4 | パスに `ノンフィクション` を含む | ノンフィクション |
| 5 | パスに `フィクション` を含む | フィクション |
| 6 | パスに `実用` を含む | 実用 |
| 7 | パスに `家庭` または `家庭関係` を含む | 家庭 |
| 8 | パスに `漫画` または `コミック` を含む | 漫画・コミック |
| 9 | パスに `コンピュータ` または `O'Reilly` を含む | コンピュータ |
| 10 | パスに `運転関係` を含む | 運転 |
| — | いずれにもマッチしない | 未分類 |

> **注意**: ノンフィクションはフィクションの部分文字列であるため、優先度4でノンフィクションを先に判定する。

**使用ジャンル一覧**（全11種）

`SF` / `フィクション（日本）` / `エッセイ` / `ノンフィクション` / `コンピュータ` / `運転` / `実用` / `家庭` / `漫画・コミック` / `フィクション` / `未分類`

### F-3: 重複排除・書籍統合

#### 概要

同一書籍の複数バージョン（オリジナル・iPad3・Kindle）を1件の書籍レコードに統合する。

#### 統合キー

| 条件 | 統合キー |
|------|---------|
| ISBNが存在する | ISBN |
| ISBNが存在しない | パース済みタイトル |

#### 統合ルール

| フィールド | ルール |
|-----------|-------|
| `title` / `author` / `series` | オリジナル版（`version: original`）の値を優先する。オリジナル版が存在しない場合は先頭レコードの値を使用 |
| `versions` | 統合された全バージョンをリストとして保持（例: `["original", "kindle"]`） |
| `genre` | ジャンル優先度テーブルに基づき、より具体的なジャンルを採用 |
| `isbn` | 非 `null` の値を優先 |
| `pages` | 非 `null` の値を優先 |

#### ジャンル優先度テーブル（重複排除時）

| 優先度 | ジャンル |
|--------|---------|
| 10 | SF |
| 9 | フィクション（日本） |
| 8 | エッセイ |
| 7 | ノンフィクション、コンピュータ、運転 |
| 6 | 実用、家庭 |
| 5 | 漫画・コミック |
| 4 | フィクション |
| 0 | 未分類 |

### F-4: books.json 生成

#### 概要

パース・ジャンル推定・重複排除後の全書籍データを `data/books.json` として出力する。

#### 出力仕様

| 項目 | 仕様 |
|------|------|
| ファイルパス | `data/books.json` |
| 文字コード | UTF-8 |
| 期待レコード数 | 819件（2026-02-27 時点） |

#### 1書籍レコードのデータ構造

```json
{
  "id": "一意識別子（ISBN または タイトルハッシュ）",
  "title": "書名",
  "author": "著者名",
  "genre": "ジャンル",
  "series": "シリーズ名 または null",
  "isbn": "ISBN または null",
  "pages": 186,
  "versions": ["original", "kindle"],
  "file_url": "https://drive.google.com/...",
  "file_id": "Google Drive ファイルID"
}
```

## 3. 書籍一覧・検索機能

### F-5: キーワード検索

#### 概要

入力したキーワードで書籍一覧をリアルタイム絞り込みする。

#### 仕様

| 項目 | 仕様 |
|------|------|
| 検索対象フィールド | `title`（タイトル）、`author`（著者名） |
| マッチ方式 | 部分一致（大文字小文字を区別しない） |
| 複数キーワード | スペース区切りで入力した場合は AND 検索 |
| 表示タイミング | 入力と同時にリアルタイム絞り込み（インクリメンタルサーチ） |
| 入力クリア | 検索フィールドのリセットボタンで全件表示に戻る |

---

### F-6: ジャンルフィルタ

#### 概要

ジャンルを選択して書籍一覧を絞り込む。

#### 仕様

| 項目 | 仕様 |
|------|------|
| 表示形式 | 全ジャンル（11種）＋「すべて」をボタンで表示 |
| 選択方式 | 単一選択 |
| 初期状態 | 「すべて」選択（全件表示） |
| キーワード検索との組み合わせ | AND 条件で絞り込む |

---

### F-7: 著者絞り込み

#### 概要

著者名をクリックまたは選択して、その著者の書籍のみ表示する。

#### 仕様

| 項目 | 仕様 |
|------|------|
| 操作方法 | 書籍一覧・書籍詳細の著者名クリック、または著者名ドロップダウン |
| 絞り込み結果 | 選択した著者の全書籍を表示 |
| 解除方法 | 「すべて」または著者名を再クリックで解除 |
| キーワード検索・ジャンルフィルタとの組み合わせ | AND 条件で絞り込む |

---

### F-8: ソート

#### 概要

書籍一覧の表示順序を切り替える。

#### 仕様

| 項目 | 仕様 |
|------|------|
| ソートキー | 書名（五十音順）、著者名（五十音順）、ページ数（昇順・降順） |
| 初期状態 | 書名の五十音順 |
| UI | ドロップダウンまたはカラムヘッダークリック |

---

### F-9: ページネーション

#### 概要

書籍一覧を複数ページに分けて表示する。

#### 仕様

| 項目 | 仕様 |
|------|------|
| 1ページあたりの表示件数 | 50件 |
| ページ切り替え | 前/次ページボタン、ページ番号直接指定 |
| 件数表示 | 絞り込み結果の総件数と現在表示範囲を表示（例: `1-50 / 819件`） |
| フィルタ変更時の挙動 | フィルタやソート変更時は1ページ目に戻る |

## 4. 書籍詳細機能

### F-10: 基本情報表示

#### 概要

書籍一覧で選択した書籍の詳細情報を表示する。基本情報はファイル名パースから取得した情報を表示する。

#### 表示項目

| 項目 | データソース | 非存在時の表示 |
|------|------------|--------------|
| タイトル | `books.json` | — |
| 著者名 | `books.json` | — |
| ジャンル | `books.json` | — |
| シリーズ名 | `books.json` | 非表示 |
| ISBN | `books.json` | 「情報なし」 |
| ページ数 | `books.json` | 「情報なし」 |
| 保有バージョン | `books.json` | — |
| Google Drive リンク | `books.json`（`file_url`） | — |

#### Google Drive リンク仕様

| 項目 | 仕様 |
|------|------|
| 表示形式 | 保有バージョンごとにリンクボタンを表示（例: 「オリジナル版で開く」「Kindle版で開く」） |
| 動作 | クリックで Google Drive の PDF を新しいタブで開く |
| 利用環境 | PC・iPad のブラウザから Google Drive へのアクセス権限が必要 |

---

### F-11: 外部書誌API補完表示

#### 概要

ISBNが存在する書籍について、外部書誌APIから追加情報を取得して詳細画面に表示する。

#### API呼び出し仕様

| 優先度 | API | 用途 | 呼び出し条件 |
|--------|-----|------|------------|
| 第1優先 | OpenBD API | 日本語書籍の書誌情報取得 | ISBNが存在する場合 |
| フォールバック | Google Books API | OpenBDでデータなしの場合（洋書対応） | OpenBDでデータが取得できなかった場合 |

> ISBNが存在しない書籍は外部API呼び出しを行わず、F-10の基本情報のみ表示する。

#### 補完表示項目

| 項目 | 取得元 | 表示条件 |
|------|--------|---------|
| 表紙画像 | OpenBD / Google Books | 取得できた場合 |
| 出版社 | OpenBD / Google Books | 取得できた場合 |
| 出版年 | OpenBD / Google Books | 取得できた場合 |
| 内容紹介・説明文 | OpenBD / Google Books | 取得できた場合 |

#### エラー・ローディング処理

| 状態 | 表示内容 |
|------|---------|
| API呼び出し中 | ローディングインジケータを表示 |
| データなし（APIから空レスポンス） | 「外部情報なし」と表示 |
| APIエラー | エラーメッセージを表示し、基本情報のみ表示継続 |

## 5. 統計・分析機能

### F-12: ジャンル別統計

#### 概要

蔵書全体のジャンル分布を視覚的に表示する。

#### 表示仕様

| 項目 | 仕様 |
|------|------|
| グラフ形式 | 棒グラフまたは円グラフ（比率表示） |
| 表示内容 | 全11ジャンルの冊数と全体に占める割合（%） |
| インタラクション | グラフのジャンルをクリックすると書籍一覧をそのジャンルで絞り込む |

---

### F-13: 著者別ランキング

#### 概要

保有冊数の多い著者のランキングを表示する。

#### 表示仕様

| 項目 | 仕様 |
|------|------|
| 表示件数 | 上位20名 |
| 表示内容 | 順位、著者名、保有冊数、代表ジャンル |
| インタラクション | 著者名をクリックすると書籍一覧をその著者で絞り込む |

## 6. 画面構成

### 画面一覧

| 画面ID | 画面名 | 主な機能 |
|--------|--------|---------|
| S-1 | 書籍一覧画面 | F-5〜F-9（検索・フィルタ・ソート・ページネーション） |
| S-2 | 書籍詳細画面 | F-10〜F-11（基本情報・API補完情報） |
| S-3 | 統計ダッシュボード画面 | F-12〜F-13（ジャンル分布・著者ランキング） |

### 画面遷移

```
書籍一覧（S-1）
  ├── 書籍カードをクリック → 書籍詳細（S-2）
  │   └── 著者名をクリック → 書籍一覧（S-1）著者絞り込み状態
  └── タブ切替 → 統計ダッシュボード（S-3）
      ├── ジャンルグラフをクリック → 書籍一覧（S-1）ジャンルフィルタ状態
      └── 著者名をクリック → 書籍一覧（S-1）著者絞り込み状態
```
