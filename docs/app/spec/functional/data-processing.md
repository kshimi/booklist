## F-1: Google Drive CSVインポート・メタデータパース

### 概要

Google Apps Script により取得した CSV ファイル（`data/booklist.csv`）を読み込み、各 PDF ファイルのメタデータを抽出する。

### 入力

| 項目 | 仕様 |
|------|------|
| ファイル形式 | CSV（Google Apps Script 出力） |
| 想定レコード数 | 約 1,900 行 |
| 必須カラム | フォルダパス、ファイル名、ファイルサイズ(MB)、MIMEタイプ、作成日、最終更新日、ファイルURL、ファイルID |

### 対象ファイルの絞り込み

- MIMEタイプ `application/pdf` のファイルのみ処理対象とする
- 以下のファイル名を含むレコードは除外する（管理用ファイル）:
  - `self_check`
  - `20120330_1246_33_0706`
  - `to_1733_912_003_5_at`

### ファイル名パース仕様

PDF ファイル名は以下の書式に概ね従う:

```
[バージョンプレフィックス_]タイトル（シリーズ名）著者名 ページ数p_ISBN.pdf
```

各フィールドを以下の順で段階的に抽出する:

| ステップ | 処理内容 |
|---------|---------|
| 1. バージョン判定 | `kindlep_` プレフィックス → Kindle版、`ipad3_` プレフィックス → iPad3版、なし → オリジナル版。プレフィックスをファイル名から除去する |
| 2. 拡張子除去 | 末尾の `.pdf` を除去 |
| 3. ISBN抽出 | 末尾の `_XXXXX` パターン（10〜13文字の英数字）を抽出・除去 |
| 4. ページ数抽出 | 末尾の ` NNNp` パターン（半角スペース + 数字 + p）を抽出・除去 |
| 5. シリーズ名抽出 | `（）` または `()` 内のテキストを全て抽出。複数ある場合は `／` 区切りで結合。括弧と内部テキストをファイル名から除去。`〈〉` などその他の括弧は処理対象外 |
| 6. 著者名抽出 | 最後の `）` または `)` の後ろに残るテキストを著者名とする。括弧がない場合は著者名を空文字列とし、後続の自動補完パターン（下記参照）にゆだねる |
| 7. タイトル抽出 | 上記ステップ後に残ったテキストをタイトルとする |

### 著者名の補完・正規化

F-1 の著者名抽出後、以下の順で著者名の補完・正規化を行う。

#### ステップ 0: 自動補完パターン（著者名が空の場合のみ）

括弧を持たないファイル名で著者名が空になったレコードに対し、タイトルテキスト末尾のパターンマッチを試みる。
いずれにも一致しない場合は著者名を空文字列のまま残し、後続の補正ファイル照合（ステップ 3）にゆだねる。

| パターン ID | 条件 | 処理 | 例 |
|------------|------|------|----|
| P-A: 欧文名 | タイトルに日本語（ひらがな・漢字・カタカナ）が含まれ、末尾が半角スペース + 欧文 Title Case 語（`[A-Z][a-z]+`）1語以上で終わる | タイトル末尾の欧文語を著者名として切り出す | `SQLアンチパターン Bill Karwin` → タイトル: `SQLアンチパターン` / 著者: `Bill Karwin` |
| P-B: 漢字氏名 | タイトルが `漢字1〜3文字 + 半角スペース + 漢字1〜2文字` で終わる（厳しめの長さ制限で偽陽性を抑制） | タイトル末尾の漢字2語を著者名として切り出す | `Schemeによる記号処理入門 猪股 俊光` → タイトル: `Schemeによる記号処理入門` / 著者: `猪股 俊光` |

> **補足**: カタカナ氏名・著/訳マーカーを含むケースは誤検知リスクが高いため自動パターンの対象外とし、補正ファイルで対応する。

#### ステップ 1: 正規化ルール

| ルール | 変換内容 | 例 |
|--------|---------|-----|
| 中黒統一 | 氏名中の半角スペース（` `）を中黒（`・`）に置換（姓 名の2単語形式を対象） | `アイザック アシモフ` → `アイザック・アシモフ` |
| 全角ドット統一 | `．` を半角ピリオド（`.`）に統一（イニシャル表記向け） | `J．R．R．` → `J.R.R.` |
| 前後空白除去 | trim 処理（既存） | — |

#### ステップ 2: エイリアステーブル照合

- 正規化ルール適用後の著者名を `data/author-aliases.json` と照合する
- マッチした場合は定義された正規名に置き換える
- マッチしない場合はルール適用後の名前をそのまま使用する
- `author-aliases.json` が存在しない場合は照合をスキップしてエラーにしない

**`data/author-aliases.json` の形式:**

```json
{
  "揺れ表記（正規化ルール適用後）": "正規名",
  "J.R.R.・トールキン": "J.R.R.トールキン"
}
```

#### ステップ 3: 補正ファイル照合（最終上書き）

正規化・エイリアス照合後に `data/book-corrections.json` と照合し、一致した書籍のタイトルと著者名を上書きする。
このステップは全ての自動処理の後に適用され、最も高い優先度を持つ。

- 照合キー: パース済みタイトル（著者名が混入した状態のタイトル）
- `book-corrections.json` が存在しない場合は照合をスキップしてエラーにしない
- 初期状態では著者名が空の書籍81件分のエントリが自動生成され、著者名フィールドは空文字列。ユーザーが著者名を記入したエントリのみ実質的に上書きされる

**`data/book-corrections.json` の形式:**

```json
{
  "corrections": [
    {
      "original_title": "GNU Emacs デブラ キャメロン",
      "title": "GNU Emacs",
      "author": "デブラ・キャメロン"
    },
    {
      "original_title": "雪国 川端 康成 208p_4101001014",
      "title": "雪国",
      "author": "川端康成",
      "pages": 208,
      "isbn": "4101001014"
    },
    {
      "original_title": "Rubyベストプラクティス",
      "title": "Rubyベストプラクティス",
      "author": ""
    }
  ]
}
```

| フィールド | 内容 |
|-----------|------|
| `original_title` | 照合に使用する元のタイトル文字列（著者名・ページ数・ISBNが混入した状態） |
| `title` | 正しい書名 |
| `author` | 正しい著者名（情報がない場合は空文字列） |
| `pages` | （省略可）正しいページ数。キーが存在する場合（`null` を含む）はパース結果を上書きする |
| `isbn` | （省略可）正しいISBN。キーが存在する場合（`null` を含む）はパース結果を上書きする |

> **補足**: `pages` / `isbn` フィールドはキーが存在しない場合にパース結果をそのまま使用する（`undefined` と `null` を区別）。Google Drive の重複ファイル（` (1)` サフィックス）でページ数・ISBNの抽出が失敗した場合、`--generate` フラグによる自動生成でタイトル文字列からこれらを検出してエントリに付与する。

---

### 出力（1レコードあたり）

| フィールド | 内容 |
|-----------|------|
| `title` | パース済みタイトル |
| `author` | 著者名 |
| `isbn` | ISBN（10桁または13桁）。なければ `null` |
| `pages` | ページ数（整数）。なければ `null` |
| `series` | シリーズ名。なければ `null` |
| `version` | `original` / `kindle` / `ipad3` |
| `folder_path` | フォルダパス（ジャンル推定に使用） |
| `file_url` | Google Drive ファイルURL |
| `file_id` | Google Drive ファイルID |
| `file_size_mb` | ファイルサイズ（MB） |
| `source` | `"google_drive"` 固定 |

## F-1b: オフライン書誌CSVインポート

### 概要

手動作成した紙書籍リスト（`data/offline_bibliography_list.csv`）を読み込み、F-1 と結合できる形式のレコードに変換する。

### 入力

| 項目 | 仕様 |
|------|------|
| ファイルパス | `data/offline_bibliography_list.csv` |
| 文字コード | UTF-8 |
| カラム | ジャンル、書名、著者名、出版社 |
| ヘッダー行 | あり（1行目） |

### 処理仕様

| 処理 | 内容 |
|------|------|
| ジャンルマッピング | 下記のマッピングテーブルに従い books.json のジャンル体系に変換する。「文学・小説・教養」は GENRE_FALLBACK_RULES で推定する |
| 著者名正規化 | F-1 と同一の正規化処理（`normalizeAuthor` → `resolveAuthorAlias`）を適用する |
| バージョン・ファイル情報 | 紙書籍のため `versions: []`、`version_files: {}` とする |
| source | `"paper"` 固定 |

#### ジャンルマッピングテーブル

| offline CSV ジャンル | books.json ジャンル | 備考 |
|--------------------|---------------------|------|
| コンピュータ・IT技術 | コンピュータ | — |
| 物理・自然科学・農学 | ノンフィクション | サブジャンルは科学・技術 |
| 文学・小説・教養 | GENRE_FALLBACK_RULES で推定 | タイトル・著者名ベース |
| コミックス | 漫画・コミック | — |
| 趣味・実用・自動車 | 実用 | 自動車系はフォールバックルールで運転に振り分け可 |

### 出力（1レコードあたり）

| フィールド | 内容 |
|-----------|------|
| `title` | 書名 |
| `author` | 著者名（正規化・エイリアス解決済み） |
| `isbn` | `null`（CSVに含まれない） |
| `pages` | `null`（CSVに含まれない） |
| `series` | `null` |
| `version` | `null` |
| `folder_path` | `null`（ジャンルマッピングテーブルで推定済み） |
| `file_url` | `null` |
| `file_id` | `null` |
| `file_size_mb` | `null` |
| `source` | `"paper"` |

## F-2: ジャンル推定

### 概要

全書籍（重複排除前の各ファイル）に対して大ジャンル（`genre`）を割り当てる。
フォルダパスによる判定で「未分類」となった書籍は、タイトル・著者名・シリーズ名のキーワードマッチで大ジャンルの補完を試みる。

### Step 1: フォルダパスによる判定（優先）

フォルダパスに含まれるキーワードを以下の優先順で判定し、最初にマッチしたジャンルを採用する。

| 優先度 | マッチ条件 | ジャンル |
|--------|-----------|---------|
| 1 | パスに `エッセイ` を含む | エッセイ |
| 2 | パスに `日本の作家` を含む | フィクション（日本） |
| 3 | パスに ` SF` または `/SF` を含む（境界条件付き） | SF |
| 4 | パスに `ノンフィクション` を含む | ノンフィクション |
| 5 | パスに `フィクション` を含む | フィクション |
| 6 | パスに `実用` を含む | 実用 |
| 7 | パスに `家庭` または `家庭関係` を含む | 家庭 |
| 8 | パスに `漫画` または `コミック` を含む | 漫画・コミック |
| 9 | パスに `コンピュータ` または `O'Reilly` を含む | コンピュータ |
| 10 | パスに `運転関係` を含む | 運転 |
| — | いずれにもマッチしない | → Step 2 へ |

> **注意**: ノンフィクションはフィクションの部分文字列であるため、優先度4でノンフィクションを先に判定する。

### Step 2: タイトル・著者名・シリーズ名による補完（フォルダパスで未分類の場合のみ）

フォルダパス判定で「未分類」となった書籍に対し、タイトル・著者名・シリーズ名のキーワードマッチで大ジャンルの補完を試みる。

| マッチ条件 | 割り当てるジャンル |
|-----------|----------------|
| SF関連キーワード（SF、宇宙人、ロボット、タイムトラベル 等） | SF |
| 漫画・コミック関連キーワード | 漫画・コミック |
| エッセイ関連キーワード | エッセイ |
| ノンフィクション関連キーワード | ノンフィクション |
| フィクション関連キーワード | フィクション |
| 上記にマッチしない | 未分類（残存） |

> **注意**: 具体的なキーワード一覧は実装フェーズで実データを確認しながら確定する。

**使用ジャンル一覧**（全11種）

`SF` / `フィクション（日本）` / `エッセイ` / `ノンフィクション` / `コンピュータ` / `運転` / `実用` / `家庭` / `漫画・コミック` / `フィクション` / `未分類`

## F-2b: サブジャンル推定

### 概要

F-2 で割り当てた大ジャンルをもとに、タイトル・著者名・シリーズ名のキーワードマッチでサブジャンル（`subgenre`）を推定する。
いずれのキーワードにもマッチしない場合は `null` とする。

> **背景**: Google Drive のフォルダ構造が現状フラットでサブジャンルフォルダが存在しないため、フォルダパスではなくタイトル等のキーワードで推定する方式を採用する（詳細は設計ドキュメント `docs/app/design/002-genre-hierarchy.md` 参照）。

### 推定対象フィールド

タイトル（`title`）・著者名（`author`）・シリーズ名（`series`）を対象とする。マッチ方式は部分一致。

### 大ジャンル別サブジャンル定義

| 大ジャンル | サブジャンル | キーワード例 |
|-----------|------------|-------------|
| フィクション | ミステリー | ミステリ、探偵、殺人、謎、推理、犯罪 |
| フィクション | 歴史・時代 | 歴史、史、帝国、革命、戦争 |
| フィクション | その他フィクション | 上記に該当しないもの |
| ノンフィクション | 歴史・伝記 | 歴史、伝記、〜の生涯、評伝 |
| ノンフィクション | 科学・技術 | 物理、数学、宇宙、進化、脳、生物、科学 |
| ノンフィクション | 社会・経済 | 経済、社会、政治、資本、投資 |
| ノンフィクション | 哲学・思想 | 哲学、思想、倫理、道徳 |
| ノンフィクション | その他ノンフィクション | 上記に該当しないもの |
| フィクション（日本） | ミステリー | ミステリ、探偵、殺人、謎、推理、犯罪 |
| フィクション（日本） | 歴史・時代 | 時代、幕末、江戸、侍、武士 |
| フィクション（日本） | その他フィクション | 上記に該当しないもの |
| 漫画・コミック | （未定） | 実装フェーズで確定 |

> **注意**: 具体的なキーワード一覧は実データを確認しながら実装フェーズで確定する。SF・エッセイ等の小規模ジャンルはサブジャンル分類の対象外（`null` のまま）。

### 出力

| フィールド | 内容 |
|-----------|------|
| `subgenre` | サブジャンル名（文字列）。判定できない場合は `null` |

## F-3: 重複排除・書籍統合

### 概要

F-1（Google Drive）と F-1b（オフライン）のレコードを結合し、同一書籍の複数バージョン（オリジナル・iPad3・Kindle）および同一書籍のデジタル・紙両方の所持を1件の書籍レコードに統合する。

### 統合キー

| 条件 | 統合キー |
|------|---------|
| ISBNが存在する | ISBN |
| ISBNが存在しない | パース済みタイトル |

### 統合ルール

| フィールド | ルール |
|-----------|-------|
| `title` / `author` / `series` | `version === 'original'` のレコードの値を優先する。オリジナル版が存在しない場合は先頭レコードの値を使用 |
| `versions` | 統合された全バージョンをリストとして保持（例: `["original", "kindle"]`）。紙書籍のみの場合は `[]` |
| `version_files` | 各バージョンの `file_url` と `file_id` をマップで保持する。紙書籍のみの場合は `{}` |
| `genre` | ジャンル優先度テーブルに基づき、より具体的なジャンルを採用 |
| `isbn` | 非 `null` の値を優先 |
| `pages` | 非 `null` の値を優先 |
| `source` | 下記の `source` 集約ルールを参照 |

### `source` 集約ルール

| 統合されたレコードの source | 出力値 |
|--------------------------|--------|
| `google_drive` のみ | `"google_drive"` |
| `paper` のみ | `"paper"` |
| `google_drive` と `paper` 両方 | `["google_drive", "paper"]` |

### ジャンル優先度テーブル（重複排除時）

| 優先度（高） | ジャンル |
|------------|---------|
| 10 | SF |
| 9 | フィクション（日本） |
| 8 | エッセイ |
| 7 | ノンフィクション、コンピュータ、運転 |
| 6 | 実用、家庭 |
| 5 | 漫画・コミック |
| 4 | フィクション |
| 0 | 未分類 |

## F-4: books.json 生成

### 概要

パース・ジャンル推定・重複排除後の全書籍データを `data/books.json` として出力する。

### 出力仕様

| 項目 | 仕様 |
|------|------|
| ファイルパス | `data/books.json` |
| 文字コード | UTF-8 |
| 期待レコード数 | 870件程度（2026-03-03 時点推定：812 + 62 - 重複分） |

### 1書籍レコードのデータ構造

```json
{
  "id": "一意識別子（ISBN または タイトルハッシュ）",
  "title": "書名",
  "author": "著者名",
  "genre": "大ジャンル",
  "subgenre": "サブジャンル または null",
  "series": "シリーズ名 または null",
  "isbn": "ISBN または null",
  "pages": 186,
  "versions": ["original", "kindle"],
  "version_files": {
    "original": { "file_url": "https://drive.google.com/...", "file_id": "Google Drive ファイルID" },
    "kindle":   { "file_url": "https://drive.google.com/...", "file_id": "Google Drive ファイルID" }
  },
  "source": "google_drive"
}
```

紙書籍のみのレコード例:

```json
{
  "id": "title_a1b2c3d4",
```
