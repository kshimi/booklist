# データパイプライン設計

**作成日**: 2026-02-28
**ステータス**: ドラフト v0.1
**対象フェーズ**: フェーズ1（静的SPA）

---

> 機能要件は [`docs/app/spec/functional/data-processing.md`](../functional/data-processing.md) を参照。
> 本ドキュメントは実装設計（HOW）を定義する。

## 1. 概要

データパイプラインはビルド前工程として Node.js スクリプト（`scripts/process.js`）で実行する。
`data/booklist.csv` を入力として受け取り、パース・ジャンル推定・重複排除を経て `data/books.json` を出力する。

```
data/booklist.csv
    │
    ├─ ステップ1: CSV読み込み・フィルタリング
    ├─ ステップ2: ファイル名パース（メタデータ抽出）
    ├─ ステップ3: ジャンル推定（フォルダパスベース）
    ├─ ステップ4: 重複排除・書籍統合
    └─ ステップ5: books.json 出力
```

---

## 2. 実行方法

```bash
node scripts/process.js
```

**前提条件**:
- `data/booklist.csv` が存在すること
- Node.js がインストール済みであること（外部ライブラリ不要）

**出力**:
- `data/books.json`（UTF-8、改行あり）
- 処理統計をコンソールに出力する（入力レコード数・除外数・出力書籍数）

## 3. ステップ別仕様

### ステップ1: CSV読み込み・フィルタリング

#### CSV形式

| カラム名 | 内容 |
|---------|------|
| フォルダパス | Google Drive上のフォルダパス |
| ファイル名 | PDFファイル名（拡張子含む） |
| ファイルサイズ(MB) | ファイルサイズ（数値） |
| MIMEタイプ | `application/pdf` 等 |
| 作成日 | 日付文字列 |
| 最終更新日 | 日付文字列 |
| ファイルURL | Google DriveのURL |
| ファイルID | Google DriveのファイルID |

#### フィルタリング条件

以下の条件を**両方**満たすレコードのみ処理対象とする:

| 条件 | 詳細 |
|------|------|
| MIMEタイプ | `application/pdf` であること |
| ファイル名に管理用識別子を含まないこと | `self_check`、`20120330_1246_33_0706`、`to_1733_912_003_5_at` のいずれも含まない |

#### 実装メモ

```javascript
const rows = parseCSV(csvText);
const filtered = rows.filter(row =>
  row.mimeType === 'application/pdf' &&
  !EXCLUDED_NAMES.some(name => row.filename.includes(name))
);
```

### ステップ2: ファイル名パース（メタデータ抽出）

ファイル名から以下のフィールドを順番に抽出する。各ステップで抽出したテキストは作業文字列から除去してから次のステップへ進む。

#### パース順序

| ステップ | 対象 | 抽出方法 |
|---------|------|---------|
| 2-1 | バージョン判定 | `kindlep_` で始まる → `kindle`。`ipad3_` で始まる → `ipad3`。なし → `original`。プレフィックスを除去する |
| 2-2 | 拡張子除去 | 末尾の `.pdf` を除去する |
| 2-3 | ISBN抽出 | 末尾の `_XXXXXXXXXX`（10〜13文字の英数字）パターンを正規表現で抽出・除去する |
| 2-4 | ページ数抽出 | 末尾の ` NNNp`（半角スペース+数字+p）パターンを正規表現で抽出・除去する |
| 2-5 | シリーズ名抽出 | `（...）` または `(...)` 内のテキストを全て抽出。複数ある場合は `／` 区切りで結合する。括弧とその内部を作業文字列から除去する |
| 2-6 | 著者名抽出 | 最後の括弧の後ろに残るテキストを著者名とする。括弧がない場合は末尾の日本語2単語（姓＋名）パターンにマッチさせる |
| 2-7 | タイトル抽出 | ステップ2-6まで処理した後の残りのテキストをトリムしてタイトルとする |

#### 正規表現定義

| フィールド | 正規表現 | 備考 |
|-----------|---------|------|
| バージョン（Kindle） | `/^kindlep_/` | ファイル名先頭 |
| バージョン（iPad3） | `/^ipad3_/` | ファイル名先頭 |
| ISBN | `/_([A-Za-z0-9]{10,13})$/` | 末尾の `_` + 英数字10〜13文字 |
| ページ数 | `/ (\d+)p$/` | 末尾の半角スペース+数字+p |
| シリーズ名 | `/[（(]([^）)]+)[）)]/g` | 全角・半角括弧どちらも対象 |

#### パース結果フィールド

| フィールド | 型 | 内容 |
|-----------|-----|------|
| `title` | string | パース済みタイトル |
| `author` | string | 著者名（トリム済み） |
| `isbn` | string \| null | ISBN。抽出できない場合は `null` |
| `pages` | number \| null | ページ数（整数）。抽出できない場合は `null` |
| `series` | string \| null | シリーズ名。抽出できない場合は `null` |
| `version` | `original` \| `kindle` \| `ipad3` | バージョン |
| `folder_path` | string | フォルダパス（元のCSVカラムをそのまま使用） |
| `file_url` | string | Google Drive URL |
| `file_id` | string | Google Drive ファイルID |
| `file_size_mb` | number | ファイルサイズ（MB） |

### ステップ3: ジャンル推定

フォルダパスのキーワードマッチにより各ファイルのジャンルを決定する。

#### 判定ルール（優先順）

| 優先度 | 条件 | ジャンル |
|--------|------|---------|
| 1 | `folder_path` に `エッセイ` を含む | エッセイ |
| 2 | `folder_path` に `日本の作家` を含む | フィクション（日本） |
| 3 | `folder_path` に ` SF` または `/SF` を含む（単語境界） | SF |
| 4 | `folder_path` に `ノンフィクション` を含む | ノンフィクション |
| 5 | `folder_path` に `フィクション` を含む（ノンフィクション以外） | フィクション |
| 6 | `folder_path` に `実用` を含む | 実用 |
| 7 | `folder_path` に `家庭` を含む | 家庭 |
| 8 | `folder_path` に `漫画` または `コミック` を含む | 漫画・コミック |
| 9 | `folder_path` に `コンピュータ` または `O'Reilly` を含む | コンピュータ |
| 10 | `folder_path` に `運転関係` を含む | 運転 |
| — | いずれにもマッチしない | 未分類 |

> **注意**: ノンフィクションはフィクションの部分文字列のため、必ずノンフィクションを先に判定すること。

---

### ステップ4: 重複排除・書籍統合

同一書籍の複数バージョン（`original` / `kindle` / `ipad3`）を1件の書籍レコードにまとめる。

#### 統合キーの決定

```
if (isbn !== null) {
  key = isbn;
} else {
  key = title;  // パース済みタイトル
}
```

#### 統合ルール

| フィールド | ルール |
|-----------|-------|
| `title`, `author`, `series` | `version === 'original'` のレコードの値を優先する。オリジナル版がない場合は最初のレコードの値を使用する |
| `versions` | 全バージョンをリストで保持する（例: `["original", "kindle"]`） |
| `genre` | 下記のジャンル優先度テーブルを参照し、最も優先度の高いジャンルを採用する |
| `isbn` | `null` でない値を優先する |
| `pages` | `null` でない値を優先する |
| `file_url`, `file_id` | オリジナル版のURLを使用する。オリジナル版がない場合は最初のレコードの値を使用する |

#### ジャンル優先度テーブル（重複排除時）

| 優先度（高） | ジャンル |
|------------|---------|
| 10 | SF |
| 9 | フィクション（日本） |
| 8 | エッセイ |
| 7 | ノンフィクション、コンピュータ、運転 |
| 6 | 実用、家庭 |
| 5 | 漫画・コミック |
| 4 | フィクション |
| 0 | 未分類 |

### ステップ5: books.json 出力

#### IDの生成

| 条件 | ID生成方法 |
|------|-----------|
| ISBNあり | `isbn` の値をそのまま使用する |
| ISBNなし | タイトル文字列の簡易ハッシュ（例: タイトルのUnicodeコードポイント合計のmod等）または連番+タイトルスラッグ |

#### 出力フォーマット（1レコード）

```json
{
  "id": "9784101020112",
  "title": "書名",
  "author": "著者名",
  "genre": "フィクション（日本）",
  "series": "シリーズ名 または null",
  "isbn": "9784101020112 または null",
  "pages": 312,
  "versions": ["original", "kindle"],
  "file_url": "https://drive.google.com/file/d/XXXXX/view",
  "file_id": "XXXXX"
}
```

#### 出力仕様

| 項目 | 仕様 |
|------|------|
| ファイルパス | `data/books.json` |
| 文字コード | UTF-8 |
| JSONフォーマット | `JSON.stringify(books, null, 2)`（インデント2スペース） |
| ソート順 | 書名の辞書順（デフォルト表示順に揃える） |

#### コンソール出力

処理終了時に以下の統計を出力する:

```
処理完了
  入力レコード数: 1,899
  除外レコード数: 10 (非PDF: 7, 管理用ファイル: 3)
  パース済みファイル数: 1,889
  統合後書籍数: 819
  出力ファイル: data/books.json
```

## 4. スクリプト実装メモ

### ファイル構成

```
scripts/
└── process.js      # メインスクリプト（単一ファイル）
```

外部ライブラリを使わず Node.js 標準モジュールのみで実装する:
- `fs` - ファイル読み書き
- `path` - パス操作

### CSVパース

Node.js 標準機能（行分割 + カンマ区切り）で実装する。
フィールド内のカンマや改行が含まれる場合はダブルクォートで囲まれているため、適切に処理する。

### エラーハンドリング

| エラー | 処理 |
|--------|------|
| `data/booklist.csv` が存在しない | エラーメッセージを出力してプロセス終了（`process.exit(1)`） |
| パースに失敗した行 | 警告ログを出力してスキップする（処理を継続する） |
| ISBN形式の不正 | 警告ログを出力し、`isbn: null` として扱う |
